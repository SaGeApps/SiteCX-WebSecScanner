Bad Authentication

x


When authentication functions related to the application are not implemented correctly, it allows hackers to compromise passwords or session ID's or to exploit other implementation flaws using other users credentials.

Attackers approach-->uses leaks or flaws in the authentication or session management functions.

example->exposed accounts,passwords,session id ti impersonate users

Why vulnerable?

1.May be captcha is not present

2.x-content type is not nosniff or not present

Prevention

1.Developers should ensure that they avoid XSS flaws that can be used to steal session IDs.

2.Where possible, implement multi-factor authentication

3.Implement weak-password checks

4.Limit or increasingly delay failed login attempts
